BODY-REACTIVE VISUALISER

MediaPipe pose tracking driving a Three.js WebGL particle system in real-time. Gold particle outlines trace detected body joints and connections. Gesture-based mode switching via finger count detection — hold up three fingers to change the visual mode.

ROLE Creator / Developer
YEAR 2025
TYPE WebGL / Computer Vision
STATUS Live

What I Built

A browser-based experience that uses your webcam as input and renders a real-time particle visualisation of your body in a Three.js WebGL scene. MediaPipe Pose runs in the browser, detecting 33 body landmarks at up to 30fps. Those landmark positions drive a gold particle system: particles cluster around joints and trace the skeletal connections, creating a luminous outline that mirrors your movement.

The interface is entirely gestural: no buttons, no clicks. The number of fingers you hold up in front of the camera switches between visual modes. Three fingers triggers one mode, five triggers another. The detection runs through MediaPipe Hands in parallel with Pose.

How It Works

👁️

MediaPipe Pose

33 body landmarks detected per frame: shoulders, elbows, wrists, hips, knees, ankles, and facial points. Each landmark has an x/y/z coordinate and a visibility score. Low-visibility landmarks are faded in the particle output rather than hidden abruptly.

✨

Gold Particle System

Each landmark spawns a cluster of particles in Three.js. The skeletal connections between landmarks are rendered as particle trails. Custom GLSL fragment shader gives the particles their warm gold colour with a soft bloom falloff. The whole system updates every frame from the pose data.

🤏

Gesture Mode Switching

MediaPipe Hands runs alongside Pose, counting extended fingers on the detected hand. A debounced gesture recogniser converts finger counts into mode change events. The visual mode switches without any UI interaction — hold up three fingers, hold for half a second, mode changes.

⚡

Real-Time Performance

Both MediaPipe models run client-side via WebAssembly. The render loop uses requestAnimationFrame. Pose inference and Three.js rendering run on separate ticks to avoid blocking the main thread during heavy frames.
